Условие
Вам дана база данных PostgreSQL с таблицей бронирований bookings. Необходимо реализовать НТТР-сервис для управления бронированиями мест.
Схема базы данных

CREATE TABLE bookings (
  id SERIAL PRIMARY KEY,
  user_id INTEGER NOT NULL,
  place_id INTEGER NOT NULL,
  time_from TIMESTAMP NOT NULL,
  time_to TIMESTAMP NOT NULL
Запуск
Зависит от выбранного языка, смотрите условие в созданном репозитории. Во всех случаях ожидается, что собранный бинарь будет принимать аргументы командной строки в
формате - -port ‹port>, где port - порт, на котором будет развернут сервис.
Ваш сервис должен реализовать следующие эндпоинты и возвращать корректные HTTP-коды:
GET /ping
Health check. Должен вернуть ТТР 200 с телом ("status": "ok").
POST /book
Создание нового бронирования.
Query параметры:
• place_id (integer) — ID места;
• user id (integer) — ID пользователя;
• from (string) — начало бронирования в формате RFC3339 (например, 2024-01-01T10: 00: eez);
• to (string) — конец бронирования в формате RFC3339.

Ответ:
• НТТР 200 — бронирование успешно создано;
• НТТР 409 — конфликт: запрашиваемый интервал пересекается с существующим бронированием для этого места.
Два полуинтервала [from1, to1) и [from2, to2) (from1 к from2) считаются пересекающимися, если from2 < to1.
GET /booklist
Получение списка бронирований.
Query параметры (один из двух):
• user_id (integer) - вернуть все бронирования пользователя:
• place_id (integer) — вернуть все бронирования места.
Ответ НТТР 200 с телом:
"bookings":
[
  "id": 1,
  "user_id": 10,
  "place id": 5,
  "from":
  *2024-01-01T16:80:602",
  "to": "2024-01-01T12:08: B8Z"
  
Бронирования должны быть отсортированы по (from, id) по возрастанию.
